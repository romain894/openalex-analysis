\hypertarget{index_autotoc_md6}{}\doxyparagraph{n\+\_\+max\+\_\+entities}\label{index_autotoc_md6}
When downloading a list of entities from the API (eg a list of works), the maximum number of entities to download. Set to None to have no limitation. This number must be a multiple of 200 (the is the number of element per page used by the library)

Default\+: {\ttfamily n\+\_\+max\+\_\+entities = 10 000}\hypertarget{index_autotoc_md7}{}\doxyparagraph{project\+\_\+datas\+\_\+folder\+\_\+path}\label{index_autotoc_md7}
Path to store the data downloaded from the API. The data will be stored as parquet files, with each file corresponding to one request.

Default\+: {\ttfamily project\+\_\+datas\+\_\+folder\+\_\+path = \char`\"{}data\char`\"{}}\hypertarget{index_autotoc_md8}{}\doxyparagraph{parquet\+\_\+compression}\label{index_autotoc_md8}
By default, the parquet files are compressed. The compression can be disabled by setting with parquet\+\_\+compression = None. For other parquet compression algorithms, see the pandas documentation.

Compressing reduces by 2 to 10 the file size while needing a negligeable time to compress or decompress. Disabling the compression is usefull if you want to read the parquet files with an external software.

Default\+: {\ttfamily parquet\+\_\+compression = \char`\"{}brotli\char`\"{}}\hypertarget{index_autotoc_md9}{}\doxyparagraph{max\+\_\+storage\+\_\+percent}\label{index_autotoc_md9}
Maximum storage usage percentage on the disk before starting to delete data stored in project\+\_\+datas\+\_\+folder\+\_\+path. The parquet file with the oldest last read data will be deleted first.

Default\+: {\ttfamily max\+\_\+storage\+\_\+percent = 95}\hypertarget{index_autotoc_md10}{}\doxyparagraph{disable\+\_\+tqdm\+\_\+loading\+\_\+bar}\label{index_autotoc_md10}
If set to True, it will disable the loading bar in the terminal output when downloading data from the Open\+Alex API.
